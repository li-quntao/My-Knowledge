# 什么是 SQL？

**SQL** (读作 "S-Q-L" 或 "Sequel") 是 **Structured Query Language**（结构化查询语言）的缩写。

简单来说，**它是人类与“数据库”沟通的标准语言**。

如果你把**数据库**想象成一个**超级强大、存放了上亿行数据的 Excel 表格**，那么 SQL 就是你用来操作这个表格的指令。你不用手动去一行行找数据，而是写一句 SQL 代码，数据库就会立刻把结果给你。

---

## 1. 它的核心作用：CRUD (增删改查)

在软件开发中，我们对数据的操作无非就是这四样，SQL 分别用四个核心命令来对应：

* **增 (Create) -> `INSERT`**
    * 指令含义："帮我往表格里加一条新用户数据。"
* **查 (Read) -> `SELECT`** (最常用)
    * 指令含义："帮我找出所有年龄小于 25 岁的用户。"
* **改 (Update) -> `UPDATE`**
    * 指令含义："把用户 A 的电话号码改成 123456。"
* **删 (Delete) -> `DELETE`**
    * 指令含义："把这几个过期的订单删掉。"

---

## 2. 它在软件架构中的位置

回顾软件的技术分类，SQL 处于**后端**与**数据库**之间：

1.  **前端**（App 界面）：通常**不直接写 SQL**。
2.  **后端**（Java/Python）：是**写 SQL 的主力**。
3.  **数据库**（MySQL/Oracle）：是**听懂并执行 SQL 的地方**。

**典型流程：**
> 用户点击“查询余额” (前端) -> 请求发送给后端 -> **后端写一句 SQL 发给数据库** -> 数据库返回数据 -> 后端展示给用户。

---

## 3. 一个直观的例子

假设你有一个表格叫 `Users` (用户表)，里面有 `Name` (名字) 和 `Age` (年龄)。

如果你想**找出所有成年人**：
* **人类语言：** “把大于 18 岁的人找出来”。
* **SQL 语言：**

```sql
SELECT Name, Age 
FROM Users 
WHERE Age >= 18;
```
SQL 的语法非常像英语，逻辑清晰直观。

---
## 4. 总结
**它是谁：** 数据库领域的标准通用语言。

**谁在用：**

* **后端程序员：**用于在软件中存取数据。

* **数据分析师：**用于从数据库里拉取数据制作报表。

* **学习难度：** 入门极简单（学会 SELECT 只要半小时），精通较难（涉及各种性能优化）。
